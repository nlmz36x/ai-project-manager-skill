# AI Project Manager Skill / 项目接管 Skill

语言 / Language：中文 | [English](README_EN.md)

一个 AI 项目接管 / AI 项目经理 Skill。

当该 Skill 被触发或用户确认启用后，AI 应先确认目标、范围、成功标准和项目规则。目标明确后，以项目经理 + 技术负责人的身份主动规划、拆分任务、调度可用子智能体、推进实现、运行验证并输出报告。

---

## 安装方式

把下面这段话发给你的 AI 编程助手，让它根据当前环境自行安装：

```text
请帮我安装这个项目接管 Skill：
https://github.com/nlmz36x/ai-project-manager-skill

要求：
1. 先检查我当前使用的 AI 编程助手支持什么安装方式。
2. 如果支持独立 Skill / 插件目录，就作为独立 Skill 安装。
3. 不要把完整 Skill 强行塞进全局规则。
4. 不要删除、覆盖、重写我已有的全局规则或项目规则。
5. 如果当前工具不支持独立 Skill，只能追加一个很短的触发说明，并且追加前必须先问我。
6. 这个 Skill 只有在我明确说“接管项目”“进入项目经理模式”“持续推进项目”“调用 project-takeover”等触发词，或我确认启用后才进入项目经理模式；普通问答和小修不要自动进入该模式。
7. 如果我的表达不够明确，必须先问我是否要调用 project-takeover Skill 进入项目经理模式。
8. 安装前如果不确定，先问我。
9. 完成后告诉我改了哪些文件、是否需要重启或新开会话。
```

如果你已经把本仓库下载到了本地，也可以把链接替换成本地路径。

---

## 推荐调用方式

```text
调用 project-takeover skill，帮我接管这个项目。
```

或者：

```text
你直接接管这个项目，但先按 project-takeover 规则确认目标、范围和风险边界。
```

如果只是模糊地说“继续推进”“帮我看看这个项目”，AI 应先询问是否启用本 Skill，而不是直接进入项目经理模式。

---

## 触发方式

明确触发示例：

- “你直接接管这个项目”
- “进入项目经理模式”
- “帮我把这个版本做完”
- “持续推进这个项目”
- “你来指挥开发”
- “你来安排子智能体”
- “按项目经理模式推进”
- “调用 project-takeover skill”

如果用户没有明确触发，只是说“继续”“帮我看看”“优化一下”“修一下”“这个项目怎么办”等模糊表达，AI 应先询问：

```text
你是想调用 project-takeover Skill，进入项目经理模式来接管并推进这个项目吗？
```

只有用户确认后，才进入项目经理模式。

普通问答、单文件小修、代码解释、简单 bug 定位、一般建议，不应自动进入该模式。

---

## 特点

- 先确认目标，不盲目执行。
- 可以质疑用户的不合理或高风险要求。
- 对破坏性、生产、部署、付费、权限或安全相关操作进行二次确认。
- 执行中遇到非阻塞不确定项先记录并继续当前目标，不频繁打断。
- 当能提升生产效率、覆盖范围或质量时，调度子智能体、子会话、并行 agent 或 workflow。
- 不局限第一个版本，也适用于 MVP、后续版本、修复、优化、重构、上线准备和长期迭代。
- 要求参考宿主环境全局规则和当前项目规则。
